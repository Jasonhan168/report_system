# 报表查询系统 — 本地部署文档

> 版本：v1.0 | 适用环境：Linux / Windows Server | 数据库：MySQL 5.7+ / MariaDB 10.3+

---

## 一、环境要求

| 组件 | 最低版本 | 推荐版本 | 说明 |
|------|---------|---------|------|
| Node.js | 18.x | 20.x LTS | 运行时环境 |
| pnpm | 8.x | 10.x | 包管理器（`npm i -g pnpm`） |
| MySQL | 5.7 | 8.0 | 也支持 MariaDB 10.3+ |
| 操作系统 | — | CentOS 7+ / Ubuntu 20.04+ / Windows Server 2019+ | — |

## 二、部署步骤

### 第一步：准备源码

将项目源码解压到服务器目录，例如 `/opt/report_query_system`。

```bash
cd /opt
unzip report_query_system.zip
cd report_query_system
```

### 第二步：安装依赖

```bash
pnpm install --frozen-lockfile
```

> 如果服务器无法访问外网，请在有网络的机器上执行 `pnpm install`，然后将整个目录（含 `node_modules`）打包上传。

### 第三步：初始化数据库

**3.1 创建数据库并建表**

登录 MySQL，执行初始化脚本：

```bash
mysql -u root -p < scripts/init-db.sql
```

或在 MySQL 客户端中执行：

```sql
SOURCE /opt/report_query_system/scripts/init-db.sql;
```

**3.2 创建专用数据库账号（推荐）**

```sql
CREATE USER 'report_app'@'localhost' IDENTIFIED BY '强密码';
GRANT ALL PRIVILEGES ON report_db.* TO 'report_app'@'localhost';
FLUSH PRIVILEGES;
```

### 第四步：配置环境变量

复制配置模板并编辑：

```bash
cp scripts/env.example.txt .env
vi .env
```

**必填配置项说明：**

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `DATABASE_URL` | MySQL 连接字符串 | `mysql://report_app:密码@localhost:3306/report_db` |
| `AUTH_MODE` | 认证模式（见下方说明） | `local` 或 `ldap` |
| `JWT_SECRET` | Session 签名密钥，生产环境必须修改 | 32位以上随机字符串 |
| `PORT` | 服务监听端口（可选，默认3000） | `3000` |

**生成强随机 JWT_SECRET：**

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

**认证模式说明：**

- `AUTH_MODE=local`：本地账号密码登录，用户名/密码存储在数据库中，适合测试和小规模内网部署。
- `AUTH_MODE=ldap`：AD 域账号登录，用户用 Windows 域账号直接登录，适合已有 AD 域的生产环境。

### 第五步：构建前端

```bash
pnpm build
```

构建完成后，前端静态文件会输出到 `dist/public/`，后端代码输出到 `dist/index.js`。

### 第六步：创建管理员账号

**仅在 `AUTH_MODE=local` 时需要此步骤。** LDAP 模式下，第一个登录的域账号默认为普通用户，需在数据库中手动将其 `role` 改为 `admin`。

```bash
node scripts/create-admin.mjs
```

按提示输入管理员用户名、显示名和密码即可。

### 第七步：启动服务

```bash
node dist/index.js
```

服务启动后，访问 `http://服务器IP:3000` 即可打开系统。

## 三、AD 域（LDAP）配置

当 `AUTH_MODE=ldap` 时，需要在系统中配置 AD 域参数。

**配置方式：** 管理员登录系统后，进入 **系统管理 → 系统配置**，填写以下参数：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| AD域服务器地址 | LDAP 服务地址，含协议和端口 | `ldap://10.10.1.1:389` |
| AD域名称 | NetBIOS 域名或完整域名 | `CORP` 或 `corp.example.com` |
| AD Base DN | 搜索用户的根路径 | `DC=corp,DC=example,DC=com` |
| AD绑定服务账号 | 可选，用于搜索用户属性（显示名、邮箱、部门） | `svc_report@corp.example.com` |
| AD绑定服务账号密码 | 绑定账号的密码 | — |

**LDAP 模式下的用户名格式（均支持）：**
- 短域名格式：`username`（系统自动补全为 `CORP\username`）
- 域名格式：`CORP\username`
- UPN 格式：`username@corp.example.com`

**首次 LDAP 登录后设置管理员：**

```sql
-- 将域账号 zhangsan 设置为管理员
UPDATE report_db.users SET role = 'admin' WHERE openId = 'zhangsan';
```

## 四、使用 PM2 守护进程（推荐生产环境）

PM2 可以在进程崩溃时自动重启，并支持开机自启。

```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start dist/index.js --name report-system

# 设置开机自启
pm2 startup
pm2 save

# 查看运行状态
pm2 status

# 查看日志
pm2 logs report-system
```

## 五、Nginx 反向代理（推荐）

如需通过 80/443 端口访问，或配置 HTTPS，建议使用 Nginx 反向代理。

```nginx
server {
    listen 80;
    server_name report.example.com;  # 替换为实际域名或IP

    location / {
        proxy_pass         http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection 'upgrade';
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 七、常见问题

**Q：启动时提示 `Cannot find module`？**
A：请先执行 `pnpm install` 安装依赖，再执行 `pnpm build` 构建。

**Q：数据库连接失败？**
A：检查 `DATABASE_URL` 格式是否正确，确认 MySQL 服务已启动，且账号有访问权限。

**Q：LDAP 登录失败，提示"AD域配置不完整"？**
A：需要先以本地账号（`AUTH_MODE=local`）登录管理后台，在系统配置中填写 AD 域参数，再切换 `AUTH_MODE=ldap` 重启服务。

**Q：忘记管理员密码？**
A：直接在数据库中更新密码哈希，或执行 `node scripts/create-admin.mjs` 创建新管理员账号。

**Q：如何升级系统？**
A：替换源码后，执行 `pnpm install && pnpm build`，然后重启服务。如有数据库变更，需同时执行新的迁移 SQL。

## 八、安全建议

在生产环境部署时，请注意以下安全事项：

1. **修改默认密钥**：`JWT_SECRET` 必须设置为强随机字符串，不可使用默认值。
2. **数据库账号权限**：为应用创建专用数据库账号，避免使用 root 账号。
3. **防火墙配置**：仅开放必要端口（如 80/443），数据库端口（3306）不应对外暴露。
4. **HTTPS 加密**：生产环境建议配置 SSL 证书，通过 HTTPS 访问系统。
5. **定期备份**：定期备份数据库，防止数据丢失。
