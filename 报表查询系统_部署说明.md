# 报表查询系统 — 部署说明文档

**版本：** v1.0  
**技术栈：** Node.js 22 · React 19 · TypeScript · tRPC · Drizzle ORM · MySQL  
**适用环境：** Linux（Ubuntu 20.04 / 22.04 推荐）

## 三、快速部署步骤

### 3.1 安装 Node.js 和 pnpm

```bash
# 安装 Node.js 22（使用 nvm）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
source ~/.bashrc
nvm install 22
nvm use 22

# 安装 pnpm
npm install -g pnpm@10
```

### 3.2 解压项目代码

```bash
tar -xzf report_query_system_code.tar.gz
cd report_query_system
```

### 3.3 安装依赖

```bash
pnpm install
```

### 3.4 配置 MySQL 数据库

```sql
-- 以 root 身份登录 MySQL，创建数据库和专用账号
CREATE DATABASE report_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'report_user'@'localhost' IDENTIFIED BY 'your_strong_password';
GRANT ALL PRIVILEGES ON report_system.* TO 'report_user'@'localhost';
FLUSH PRIVILEGES;
```

### 3.5 配置环境变量

在项目根目录创建 `.env` 文件：

```bash
cat > .env << 'EOF'
# 数据库连接（必填）
DATABASE_URL=mysql://report_user:your_strong_password@localhost:3306/report_system

# JWT 会话密钥（必填，建议使用随机字符串）
JWT_SECRET=your_jwt_secret_at_least_32_chars_long

# 应用 ID（用于 OAuth，本地部署可填任意字符串）
VITE_APP_ID=report_query_system_local

# OAuth 服务地址（本地部署时指向自身）
OAUTH_SERVER_URL=http://localhost:3000
VITE_OAUTH_PORTAL_URL=http://localhost:3000

# 系统所有者信息（首次登录的用户将自动成为管理员）
OWNER_OPEN_ID=admin
OWNER_NAME=系统管理员

# 内置 API（本地部署可留空或填占位符）
BUILT_IN_FORGE_API_URL=http://localhost:3000
BUILT_IN_FORGE_API_KEY=local_key
VITE_FRONTEND_FORGE_API_KEY=local_key
VITE_FRONTEND_FORGE_API_URL=http://localhost:3000
VITE_ANALYTICS_ENDPOINT=
VITE_ANALYTICS_WEBSITE_ID=
EOF
```

> **安全提示：** 生产环境中请将 `.env` 文件权限设置为 `600`，并确保不将其提交到版本控制系统。

### 3.6 执行数据库迁移

```bash
# 生成迁移 SQL 并应用到数据库
pnpm drizzle-kit migrate
```

迁移完成后，数据库将自动创建以下表：

| 表名 | 说明 |
|------|------|
| `users` | 用户账号（含角色、部门、启用状态） |
| `datasources` | 数据源连接配置 |
| `report_modules` | 报表模块注册信息 |
| `report_permissions` | 用户报表权限（查看/导出） |
| `system_configs` | 系统全局配置（AD域、会话超时等） |

### 3.7 构建前端资源

```bash
pnpm build
```

构建产物输出到 `dist/` 目录。

### 3.8 启动服务

**开发模式（热重载）：**
```bash
pnpm dev
```

**生产模式：**
```bash
pnpm start
```

服务默认监听 `http://localhost:3000`。

## 五、使用 Nginx 反向代理（推荐生产环境）

安装 Nginx 并创建配置文件：

```bash
sudo apt install nginx -y
sudo nano /etc/nginx/sites-available/report-query-system
```

写入以下配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为实际域名或 IP

    # 将 HTTP 重定向到 HTTPS（如已配置 SSL）
    # return 301 https://$host$request_uri;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
    }
}
```

```bash
# 启用站点并重启 Nginx
sudo ln -s /etc/nginx/sites-available/report-query-system /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 八、接入真实数据库（替换 Mock 数据）

当前版本使用 Mock 数据演示，接入真实数据库需修改 `server/routers.ts` 中的查询逻辑：

### 8.1 接入 MySQL

```bash
# 已内置 mysql2 驱动，无需额外安装
```

在 `server/routers.ts` 的 `pkgWipSummary.query` 过程中，将 `generateMockWipData()` 替换为实际 SQL 查询：

```typescript
import { getDb } from "./db";

// 替换 Mock 查询
const db = await getDb();
const rows = await db.execute(sql`
  SELECT
    label_name    AS labelName,
    vendor_part   AS vendorPartNo,
    vendor_name   AS vendorName,
    qty_unissued  AS qtyUnissued,
    qty_die_bond  AS qtyDieBond,
    qty_wire_bond AS qtyWireBond,
    qty_molding   AS qtyMolding,
    qty_test      AS qtyTest,
    qty_post_test AS qtyPostTest
  FROM pkg_wip_summary
  WHERE report_date = ${input.date}
    AND (${input.labelName} = '' OR label_name = ${input.labelName})
    AND (${input.vendorName} = '' OR vendor_name = ${input.vendorName})
  ORDER BY label_name, vendor_name
  LIMIT ${input.pageSize} OFFSET ${(input.page - 1) * input.pageSize}
`);
```

### 8.2 接入 ClickHouse

```bash
pnpm add @clickhouse/client
```

```typescript
import { createClient } from "@clickhouse/client";

const client = createClient({
  host: process.env.CLICKHOUSE_HOST,
  username: process.env.CLICKHOUSE_USER,
  password: process.env.CLICKHOUSE_PASSWORD,
  database: process.env.CLICKHOUSE_DB,
});

const result = await client.query({
  query: `SELECT * FROM pkg_wip_summary WHERE report_date = {date:String}`,
  query_params: { date: input.date },
  format: "JSONEachRow",
});
```

### 8.3 接入 Oracle

```bash
pnpm add oracledb
```

```typescript
import oracledb from "oracledb";

const connection = await oracledb.getConnection({
  user: process.env.ORACLE_USER,
  password: process.env.ORACLE_PASSWORD,
  connectString: process.env.ORACLE_CONNECT_STRING,

const result = await connection.execute(
  `SELECT * FROM PKG_WIP_SUMMARY WHERE REPORT_DATE = :date`,
  { date: input.date }
```

## 十、新增报表模块

系统采用模块化架构，新增报表只需三步：

**第一步：** 在数据库 `report_modules` 表中注册新模块：

```sql
INSERT INTO report_modules (code, name, description, category, route, icon, status, sort_order)
VALUES ('new_report_code', '新报表名称', '报表描述', '分类名称', '/reports/new-report', 'FileText', 'active', 2);
```

**第二步：** 在 `server/routers.ts` 中添加新报表的 tRPC 路由（参考 `pkgWipSummary` 的实现模式）。

**第三步：** 在 `client/src/pages/` 下创建对应页面组件，并在 `client/src/App.tsx` 中注册路由。

## 十三、常见问题排查

| 问题现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 启动报 `DATABASE_URL` 错误 | 环境变量未配置 | 检查 `.env` 文件是否存在且格式正确 |
| 数据库迁移失败 | MySQL 用户权限不足 | 确认用户拥有 `CREATE TABLE` 权限 |
| 页面显示"没有权限" | 用户权限未分配 | 管理员在「报表权限」页面分配权限 |
| 登录后跳转失败 | OAuth 回调地址不匹配 | 检查 `OAUTH_SERVER_URL` 与实际访问地址一致 |
| Excel 导出为空 | 未查询数据直接导出 | 先点击「查询」按钮加载数据，再导出 |
| 端口 3000 被占用 | 其他进程占用 | 修改 `server/_core/index.ts` 中的端口号，或 `kill` 占用进程 |

## 十四、安全建议

在生产环境部署时，请务必注意以下事项：

- `JWT_SECRET` 应使用至少 32 位的随机字符串，可通过 `openssl rand -base64 32` 生成。
- 数据库密码应足够复杂，且数据库端口（默认 3306）不应对外网开放。
- 建议配置 HTTPS，避免用户凭证在传输过程中被截获。
- 定期备份数据库，尤其是 `users`、`report_permissions` 等核心表。
- 生产环境中应将 `NODE_ENV` 设置为 `production`，以启用安全相关的中间件配置。
